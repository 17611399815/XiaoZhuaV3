## 1. 项目愿景与定位

“小爪 (XiaoZhua)” 是一款专为新生代宠物主打造的**全场景、暖色治愈风宠物管家与智能硬件联调平台**。应用深度融合了“日常记账、健康追踪、智能设备互联、AI 智能顾问、宠物囤货管理、SPA预约及成长相册”等核心板块，其设计语言围绕高饱和暖橘色调（#FFB23F）展开，旨在为用户提供有温度的、沉浸式的养宠体验。

### 核心定位
*   **多端原生体验**：基于 Flutter 3.x 研发，实现 iOS 17+、Android 10+ 以及 **HarmonyOS NEXT (纯血鸿蒙)** 的三端高保真、高性能原生运行。
*   **混合架构双轨并存**：移动端完全采用 Flutter 独立原生开发，同时在管理后台内置高保真 H5 仿真模拟器作为沙箱，实现 H5 小程序与原生代码的平行联调。
*   **AI 赋能健康管理**：直连 **Gemini 3.5-flash** 极速推理引擎，结合本地意图分类器，根据宠物病史、体重、年龄进行定制化健康辅导。

```
小爪 App 混合云架构
├── mobile (Flutter 客户端)      --> 运行于 iOS/Android/鸿蒙原生环境，通过 RESTful 与服务端通信
```

## 4. 移动端 (Flutter) 核心架构与业务

Flutter 客户端代码高度模块化，通过 `Provider` 集中式状态机将 UI 视图层与网络/存储层剥离。

### 4.1 状态管理与数据总线 (`AppProvider`)
位于 `mobile/lib/services/app_provider.dart`，负责：
*   **登录会话状态**：保存当前操作的用户信息、JWT 令牌。
*   **宠物多档管理**：支持多只宠物建档，通过 `switchPet(Pet)` 方法实现一键切换首页画像，所有依赖页面响应式重绘。
*   **建档向导状态流**：托管 5 步建档向导的临时实体 `_pendingPet`，确保中途退出不会导致数据碎片。
*   **业务数据集**：内存同步管理“提醒、记账、食谱、日记、体重、病历、囤货、商城购物车、智能设备列表”。
*   **数据填充**：提供 `seedDemoData()` 快速向当前账户注入全套极具真实感的养宠测试数据。

### 4.2 路由总线设计 (`main.dart`)
移动端通过原生命名路由 `onGenerateRoute` 进行精确过渡，避免硬编码：
*   `/` ：**手机验证登录页**。支持倒计时、一键填充快捷手机。
*   `/profile-setup` ：**5步宠物建档流程**。
*   `/home` ：**主导航控制器 (MainNavigator)**。托管 5 个核心底部 Tab：
    1.  **首页 (`HomePage`)**：信息密度极高，包含宠物专属徽章、3 状态胶囊条、SPA 滚动横幅、智能设备状态板、12 宫格快捷菜单。
    2.  **商城 (`ShopPage`)**：商品分类筛选、购物车悬浮窗、积分抵扣、一键下单。
    3.  **AI管家 (`AiAssistantPage`)**：沉浸式 AI 聊天，底栏圆圈突起设计，支持多轮宠物健康会话。
    4.  **相册 (`AlbumPage`)**：时间轴瀑布流、分类相册、上传本地照片。
    5.  **个人中心 (`ProfilePage`)**：宠物生命周期统计、常用工具快捷菜单、退出登录。
*   `/message` ：**消息中心**。设备警告、囤货不足提醒、商城物流、系统通知。
*   `/device` ：**智能硬件管理**。在线状态监测、智能喂水器绑定、自动投食器余量显示、定位项圈轨迹。
*   `/spa` ：**高端 SPA 预约**。针对不同宠物品种推荐夏日草本深层芳疗服务，集成预约时间槽和预约记录。
*   `/pet-edit` ：**宠物信息精细化编辑**。允许修改品种、生日、性别、绝育状态、过敏史及用药历史。

## 6. 后端服务与 AI 意图引擎

### 6.1 联调开发后端 (`server.ts`)
基于 Express 框架快速建立的 mock API 服务。当运行 `npm run dev` 时，它会作为后台主进程启动：
*   **RESTful API 模拟**：提供包括 `/api/auth/send-code`、`/api/auth/login`、`/api/pets`、`/api/expenses`、`/api/ai/chat` 等 30+ 组符合生产规范的 API 路由。
*   **本地内存状态机**：所有通过模拟器修改、添加的数据（如新增消费记录、上传照片、更改宠物体重）均会发送到后端，并在内存中进行数据更新与响应，实现真机联调的实时反馈。

### 6.2 智能 AI 意图分类器与 Prompt 工程
在 `/api/ai/chat` 接口中，融合了精细的本地预处理逻辑：
1.  **分级意图分类器**：
    *   **强宠物相关**（如“猫咪拉肚子怎么办”、“狗粮推荐”）：通过 API 转发给 **Gemini 3.5-flash** 进行专业宠物医疗、喂养常识解答。
    *   **弱宠物相关 / 闲聊**：以温和可爱的猫狗口吻进行互动，适当引导用户补充宠物画像。
    *   **非宠物相关**（如“写一段 Java 代码”）：AI 将礼貌拒绝，维护“宠物管家”的角色专注度。
    *   **紧急词触发**（如“中毒”、“抽搐”、“骨折”）：在回复顶部红色加粗显示**紧急就医避险提示**。
2.  **动态宠物画像拼接 (Profile Injection)**：
    *   每次向 AI 发起请求时，Express 服务端会自动从内存中提取当前宠物的**年龄、品种、体重、病史、绝育状况**，将这些上下文无缝拼接至 Prompt 头部，从而使 Gemini 产生的每一条健康分析都是“因宠制宜”的定制方案。
3.  **智能兜底机制**：
    *   若网络波动导致 API 调用失败，后台会自动触发智能降级，返回极具亲和力的“猫咪助手”本地兜底文本，避免抛出底层技术错误。

## 7. 数据库设计 (Prisma Schema)

生产环境下使用 **PostgreSQL** 作为核心关系型数据库，通过 Prisma 统一建模。共设计 14 张表，保持高度的第三范式关联与级联删除：

```prisma
// backend/prisma/schema.prisma

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

// 1. 用户表
model User {
  id           String        @id @default(uuid())
  phone        String        @unique
  nickname     String?
  avatarUrl    String?
  createdAt    DateTime      @default(now())
  updatedAt    DateTime      @updatedAt
  pets         Pet[]
  orders       Order[]
  refreshTokens RefreshToken[]
}

// 2. 宠物档案表
model Pet {
  id             String          @id @default(uuid())
  userId         String
  user           User            @relation(fields: [userId], references: [id], onDelete: Cascade)
  name           String
  avatar         String          // 保存本地资源路径或 OSS 链接
  type           String          // CAT, DOG, RABBIT 等
  breed          String          // 品种
  gender         String          // GG, MM
  isSterilized   Boolean         @default(false)
  birthday       DateTime
  weight         Float           // 最新体重 (kg)
  allergies      String?         // 过敏史
  medicalHistory String?         // 过往病史
  createdAt      DateTime        @default(now())
  updatedAt      DateTime        @updatedAt

  reminders      Reminder[]
  expenses       Expense[]
  notes          Note[]
  weightRecords  WeightRecord[]
  medicalRecords MedicalRecord[]
  albumPhotos    AlbumPhoto[]
}

// 3. 疫苗与驱虫提醒表
model Reminder {
  id          String   @id @default(uuid())
  petId       String
  pet         Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  title       String   // 比如：打妙三多、体内驱虫
  type        String   // VACCINE, DEWORM, DIET, CUSTOM
  dueDate     DateTime // 预定执行时间
  isCompleted Boolean  @default(false)
  createdAt   DateTime @default(now())
}

// 4. 记账消费表
model Expense {
  id          String   @id @default(uuid())
  petId       String
  pet         Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  amount      Float    // 消费金额
  category    String   // FOOD, TOY, MEDICAL, BEAUTY, STOCK, OTHER
  remark      String?  // 备注
  date        DateTime @default(now())
}

// 5. 记事/成长日记表
model Note {
  id        String   @id @default(uuid())
  petId     String
  pet       Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  title     String
  content   String
  date      DateTime @default(now())
}

// 6. 体重历史表
model WeightRecord {
  id        String   @id @default(uuid())
  petId     String
  pet       Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  weight    Float
  date      DateTime @default(now())
}

// 7. 病历与就诊卡表
model MedicalRecord {
  id          String   @id @default(uuid())
  petId       String
  pet         Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  diagnose    String   // 诊断结果
  hospital    String?  // 就诊医院
  treatment   String   // 治疗与用药建议
  date        DateTime @default(now())
}

// 8. 囤货库存表
model StockItem {
  id        String   @id @default(uuid())
  petId     String
  name      String   // 商品名称 (如：巅峰猫粮、网易严选猫砂)
  category  String   // DIET, TOILET, HEALTH, OTHER
  current   Float    // 当前剩余 (如 2.5 kg)
  total     Float    // 总量上限 (如 10.0 kg)
  unit      String   // 单位 (kg, 包, 罐, 盒)
  updatedAt DateTime @updatedAt
}

// 9. 共享成长相册表
model AlbumPhoto {
  id        String   @id @default(uuid())
  petId     String
  pet       Pet      @relation(fields: [petId], references: [id], onDelete: Cascade)
  url       String
  caption   String?  // 相片描述
  createdAt DateTime @default(now())
}

// 10. 商城商品表
model Product {
  id          String   @id @default(uuid())
  name        String
  price       Float
  originalPrice Float?
  image       String   // 图片链接或本地矢量图标
  stock       Int      @default(99)
  sales       Int      @default(0)
  description String?
  category    String   // DIET, TOY, MEDICINE, Daily
}

// 11. 购物车记录表
model CartItem {
  id        String   @id @default(uuid())
  userId    String
  productId String
  quantity  Int      @default(1)
}

// 12. 交易订单表
model Order {
  id          String      @id @default(uuid())
  userId      String
  user        User        @relation(fields: [userId], references: [id], onDelete: Cascade)
  totalAmount Float
  status      String      // PENDING, PAID, SHIPPED, COMPLETED, CANCELLED
  createdAt   DateTime    @default(now())
  items       OrderItem[]
}

// 13. 订单子明细表
model OrderItem {
  id        String   @id @default(uuid())
  orderId   String
  order     Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)
  productId String
  price     Float
  quantity  Int
}

// 14. 令牌表与管理员表
model RefreshToken {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  token     String   @unique
  expiresAt DateTime
}

model Admin {
  id        String   @id @default(uuid())
  username  String   @unique
  password  String
  role      String   @default("OPERATOR") // SUPERADMIN, OPERATOR
}
```

## 8. 调试与部署指南

### 8.1 快速本地开发环境部署 (推荐)
仅需一步，即可并行跑通 React 管理后台、H5 沙箱、Express 后端和 Mock API 服务：
```bash
# 1. 在根目录下安装基础套件
npm install

# 2. 启动前端 + 沙箱 + 后端网关 (端口 3000)
npm run dev
```
打开浏览器访问 `http://localhost:3000` 即可开始联调。

### 8.2 原生 Flutter 客户端调试
在保证上述后端服务开启的状态下，进入 `mobile` 目录执行：
```bash
cd mobile

# 获取 Flutter 依赖包
flutter pub get

# 代码静态质量分析
flutter analyze

# 在 Chrome 中运行预览版 WebAPP
flutter run -d chrome

# 在 iOS/Android 真机或模拟器上启动原生端
flutter run -d <device_id>
```

### 8.3 纯血鸿蒙 (HarmonyOS NEXT) 编译注意事项
小爪 App 已在 `mobile` 工程内预置了针对鸿蒙原生多端编译的支持：
1.  确保本地已配置 `HUAWEI DevEco Studio` 环境及专属 SDK。
2.  通过华为 Flutter 编译工具链，绑定 `flutter_ohos` 引擎。
3.  在控制台执行：
    ```bash
    flutter build hap --release
    ```
4.  将打包出的 `.hap` 文件直接安装至鸿蒙系统。

## 10. 接手研发与后续迭代路线

### 10.1 新人接手第一步
1.  **首看路由**：直接在 `mobile/lib/main.dart` 顺藤摸瓜，看整个 Flutter 原生端的页面是如何注册并组织交互状态的。
2.  **次看状态**：阅读 `mobile/lib/services/app_provider.dart`，了解在原生端进行“添加提醒”、“新增记账”及“宠物多档切换”时，状态是如何通过 `notifyListeners()` 冒泡传递的。
3.  **调试 AI**：在 `server.ts` 中的 `app.post("/api/ai/chat", ...)` 查看画像拼接 Prompt 的逻辑，以此来了解如何添加新的上下文规则。

### 10.2 下一步迭代建议
1.  **物理真机硬件联调**：目前 `pages/device/` 仅为接口模拟，后续可考虑使用 `flutter_blue_plus` (蓝牙) 或 MQTT 插件，真正连接现实中的宠物饮水机和定位项圈。
2.  **离线持久化**：用 `hive` 或 `sqflite` 替换当前移动端内存状态，确保原生端在无网、离线环境下，依然能够秒级加载首屏。
3.  **鸿蒙原生 API 对接**：在鸿蒙 NEXT 平台下，进一步编写平台信道（Platform Channel），对接华为服务卡片（Service Widget），实现不打开 App 也能在手机桌面上看到“妙三多疫苗接种提醒”的桌面小部件。
